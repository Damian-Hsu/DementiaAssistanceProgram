開發規範說明 
1. API 文件 每次開發任務請依照 api_docs.md 格式生成與對應。 
2. 技術棧 JavaScript、HTML、CSS、Python (Flask) 僅允許使用上述語言與框架。 
3. 程式設計風格 採用 物件導向開發。 功能模組化，方便後續修改與維護。 HTML 中不直接寫 CSS/JS；CSS 集中於 style.css（後續視情況再拆分）。 新增/修改API一律寫於APIClient.js集中管理，你可以依照文件所提供的類別提升為class 
4. 命名規範 變數、方法需直觀易懂，符合語意。 關於變數、元件方面若進行程式/邏輯上的修改，請參考現有名稱。
5. 認證機制 登入/登出依照是否持有 JWT Token 判斷。 可參考 AuthService.js 的既有方法。 
6. 介面與使用流程 頁面類型： 登入 註冊 聊天 使用者資訊 事件資訊 影片資訊（含串流） 設定 流程規則： 使用者未登入 → 只能看到「登入」「註冊」。 成功登入或註冊後 → 自動切換至功能介面（不可再看到登入/註冊介面）。 登出後 → 回到登入/註冊介面。 設計風格：介面簡潔、操作直覺、不過度花俏。 
7. JWT 規範 JWT 驗證、Token 管理一律由後端處理。 前端只需接收並傳遞，不得自行生成或驗證。 
8. 程式執行方式 專案執行一律透過 app.py 啟動。 不允許： 生成執行檔 自行產生/驗證 JWT Token 建立測試帳號/密碼 
9. 任務提交要求 完成任務時需明確說明： 修改了哪個檔案 修改了哪個區塊/功能
10. 不允許試圖新增API
